The Need For Clustering

  • There might be natural variations in data.
    • Weekend vs. weekday spending patterns.
    • Heart rhythms at rest vs. during sport.
  • Enter: Cluster methods.
20

Clustering 101

  • Needed: Some measure for distance between measurements.
    • I.e. a metric.
  • Objective: Find kk clusters of points that are close together.
    • For some value of 'close'.
    • Some methods find kk automatically.
    • Most methods need it as input.
  • Now identify points not fitting in.

Anomalies as prediction error.

  • Build a model that predicts y(t)≈f(x)y(t)≈f(x).
  • E.g. using autoregressive modeling.
    • x(i)(t)=y(t−Δi)x(i)(t)=y(t−Δi).
  • Estimate the mean modeling error μ=E[y−f]≈0μ=E⁡[y−f]≈0.
  • Estimate the variation σ2=E[(y−f−μ)2]σ2=E⁡[(y−f−μ)2].
  • Calculate a zz score.
  • Flag anomalies if z>zmaxz>zmax.

Peer Group Analysis

  • Used for e.g. time series data.
  • Define train period.
  • Find parts of time series with similar behavior.
  • Group those together.
  • Calculate in-group z−scorez−score.
  • Very similar to kk-nearest neighbors.
